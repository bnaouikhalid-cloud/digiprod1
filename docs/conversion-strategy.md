# Conversion Strategy

## Conversion objective

Move a non-technical business owner through four mental states:

1. **Curious** — “This understands the tools my team already uses.”
2. **Problem-aware** — “This describes the delay, chasing and dependency I already feel.”
3. **Risk-reassured** — “We can begin without disrupting the whole team.”
4. **Ready to enquire** — “A process workshop is the sensible next decision.”

## Shared narrative

`Felt friction → why ordinary software fails → familiar-tool integration → business control → proof → implementation confidence → objection removal → enquiry`

## Page roles

### Home

- Names the operational pain in founder language.
- Introduces Flexcel as the missing operational layer.
- Shows the wider DigiProd service ecosystem as the next logical layer.
- Converts to an automation opportunity or process workshop.

### Flexcel

- Makes before/after integration understandable in seconds.
- Shows what remains familiar at each implementation stage.
- Translates ten practical use cases from pain to outcome.
- Converts to a workflow-fit discussion.

### Services

- Makes Flexcel the operational foundation, not a disconnected product mention.
- Explains when strategic guidance, execution or custom tools are needed.
- Organizes nine service categories by business outcome.
- Converts to a technology-roadmap discussion.

## CTA architecture

The site uses one destination with many contextual prompts. Each prompt sets the hidden CTA context and preselects the appropriate service/challenge when possible.

Scroll-depth pattern:

- Header: `Enquire Now`
- Hero: page-specific primary conversion
- Pain/problem section: diagnostic CTA
- Integration/outcome section: fit CTA
- Proof section: similar-use-case CTA
- Implementation section: workshop CTA
- FAQ: specific-question CTA
- Final: full enquiry form
- Mobile: persistent contextual enquiry bar

No CTA is dead. Buttons either navigate to a valid local route, open a valid external DigiProd route, call/email, or scroll to `#enquiry`.

## Trust strategy

- Preserve exact testimonial wording rather than rewriting customer meaning.
- Keep `Requires Client Approval` labels visible until identity and outcome approval is supplied.
- Do not use anonymous avatar photography.
- Avoid all unsupported ROI percentages, guarantees and security superlatives.
- Clarify that the static form demonstrates behavior but does not transmit data.
- Phrase data/IP ownership as current-site positioning that must be confirmed contractually.

## Form strategy

Fields:

- Name
- WhatsApp number
- Work email
- Company
- Business size
- Primary challenge
- Help type
- Optional message
- Consent
- Hidden source page
- Hidden CTA context

Behavior:

- Persistent labels and correct autocomplete hints.
- Inline error messages and invalid states.
- Loading state on valid submission.
- Accessible success message.
- CTA-driven preselection.
- No external transmission in the static prototype.

Production requirement: connect the validated payload to DigiProd’s approved secure lead endpoint and replace the transparent prototype success text only after successful server confirmation.
